    - **最少接続数 (Least Connection)**: 現在の接続数が最も少ないサーバーに振り分ける。
    - **重み付けラウンドロビン/最少接続数**: 各サーバーの性能に応じて重み付けを行い、優先度を考慮して振り分ける。

- **SSL/TLS終端 (SSL/TLS Offload)**: ロードバランサなどの装置が、クライアントとの間のSSL/TLS通信を終端し、バックエンドのサーバーにはHTTPで通信を転送する機能。これにより、バックエンドサーバーの負荷を軽減できる。

**関連する重要な技術知識・用語の補足説明**:
- **RTBH (Remote Triggered Black Hole) Filtering**: BGPの経路広告を利用して、特定の宛先IPアドレスへのトラフィックを、ネットワークのエッジルータで破棄させるDDoS対策技術。
- **HTTPS Flood攻撃**: HTTPSリクエストを大量に送信し、WebサーバーやSSL/TLS終端装置に過剰な負荷をかけるDDoS攻撃の一種。

#### 設問3(2)
設問の内容を簡潔に示す: RTBH方式と比較したBGP Flowspec方式の長所。

> **正解**: より細かい条件で選別して破棄することができる。

**解答に至るための思考プロセス**:
問題文には「BGP Flowspec 方式では, DDoS 検知サーバからのiBGP ピアリングで, DDoS 攻撃の宛先IPアドレスだけではなく, DDoS攻撃の送信元 IPアドレス、宛先ポート番号などを組み合わせて BGP ルータに広告して該当の通信をフィルタリングすることができる。」とあります。
- **RTBH方式**: 基本的に宛先IPアドレスのみを対象とし、その宛先IPアドレスへの全てのトラフィックを遮断します。これは攻撃対象への正当な通信も巻き込む可能性があります。
- **BGP Flowspec方式**: 宛先IPアドレスに加えて、**送信元IPアドレス、宛先ポート番号、プロトコルタイプ**など、より詳細なパケットヘッダ情報を組み合わせてフィルタリングルールを定義し、BGPで広報できます。これにより、正当な通信は許可しつつ、攻撃トラフィックのみを狙って破棄するといった、**きめ細やかな対策**が可能になります。

**関連する重要な技術知識・用語の補足説明**:
- **BGP Flowspec (RFC 8955)**: BGPを利用して、より詳細なトラフィックフィルタリングルール（フロー仕様）をネットワーク全体に配布するDDoS対策技術。

**解答に至るための思考プロセス**:
`isInNet(ip, networkAddress, subnetMask)` は、`ip` が `networkAddress` と `subnetMask` で定義されるネットワークに属するかどうかを判定する関数です。

- サブネットマスク: `255.240.0.0`

`255.240.0.0` を2進数で表すと `11111111.11110000.00000000.00000000` となり、これは `/12` のサブネットマスクに相当します（先頭から12ビットがネットワーク部）。
したがって、ネットワークアドレス `172.16.0.0/12` の範囲は以下のようになります。
- 最初のIPアドレス: ネットワークアドレスそのものなので **172.16.0.0**
- 最後のIPアドレス: ネットワークアドレスのネットワーク部を保持し、ホスト部（残りの20ビット）を全て1にしたアドレスです。
    - `172.16.0.0` の2進数（先頭12ビット）: `10101100.0001`
    - ホスト部を全て1にする: `10101100.0001 11111111.11111111.11111111`
    - これは10進数で `172.31.255.255` となります。

**関連する重要な技術知識・用語の補足説明**:
- **サブネットマスク**: IPアドレスのネットワーク部とホスト部を区別するための32ビットの数値。
- **CIDR (Classless Inter-Domain Routing)**: IPアドレスのネットワーク部を可変長で表現する方式。`/12` は先頭12ビットがネットワーク部であることを示す。

#### 設問3(1)イ～オ
設問の内容を簡潔に示す: WPADに関する空欄補充。

> **正解**:
> イ: **DHCP**
> ウ: **DNS**
> エ: **HTTP**
> オ: **URL**

**解答に至るための思考プロセス**:
WPAD (Web Proxy Auto-Discovery) は、PCやWebブラウザが自動的にPACファイルの場所を発見するためのプロトコルです。
- WPADは、**DHCP**オプションや**DNS**の名前解決（WPADというホスト名）を利用して、PACファイルの場所を通知します。
- PCやWebブラウザは、WPADを有効にしていると、これらの方法でPACファイルの場所（**URL**）を取得します。
- そして、取得したURLに対して**HTTP**リクエストを送信し、PACファイルをダウンロードしようとします。

問題文の記述と照らし合わせます。
- 「WPADは、**イ** や **ウ** の機能を利用して、PAC ファイルの場所を配布するプロトコルである。」 → **イ: DHCP**、**ウ: DNS**
- 「PCやWeb ブラウザの Web プロキシ自動検出が有効になっていると、**イ** サーバや **ウ** サーバと通信を行い、アプリケーションレイヤープロトコルの一つである **エ** を利用して **エ** サーバから PAC ファイルのダウンロードを試みる。」 → **エ: HTTP** (2箇所同じ記号なので同じ回答)
- 「PCやWeb ブラウザにはPAC ファイルの **オ** を直接設定する。」 → **オ: URL** (WPADを利用しない場合、PACファイルのURLを直接設定する)

**関連する重要な技術知識・用語の補足説明**:
- **WPAD (Web Proxy Auto-Discovery Protocol)**: Webブラウザがプロキシ設定を自動的に検出するためのプロトコル。DHCPオプション252やDNSの名前解決（wpad.example.com）を利用して、PACファイルのURLをクライアントに通知する。
- **PACファイル (Proxy Auto-Configuration file)**: プロキシサーバの利用ルールを記述したJavaScript形式のファイル。

#### 設問3(2)
設問の内容を簡潔に示す: 下線⑦（WPADの脅威）の種類。

> **正解**: 不正なプロキシサーバに中継される。

**解答に至るための思考プロセス**:
問題文の下線⑦は「悪意のある イ サーバや ウ サーバがあると⑦ PC や Web ブラウザが脅威にさらされる可能性も指摘されている。」とあります。
WPADは、DHCPやDNSの応答を偽装したり、WPADファイルを不正に配置したりすることで、悪意のあるプロキシサーバにクライアントの通信を誘導する攻撃が可能です。これにより、クライアントのWebトラフィックが正規のプロキシではなく、**不正なプロキシサーバに中継**されてしまい、通信内容の盗聴や改ざん、フィッシングサイトへの誘導などの脅威にさらされる可能性があります。

**関連する重要な技術知識・用語の補足説明**:
- **WPAD攻撃**: WPADの自動検出機能を悪用し、クライアントのトラフィックを攻撃者が用意したプロキシサーバに転送させる攻撃。中間者攻撃の一種。
