# 問1 解説

## 問題の概要
本問題は、A社が開発した**パワーアシストスーツ（PAスーツ）**に関するものです。物流倉庫などで働く作業者の腰や下肢の負担を軽減することを目的とし、各種センサー（IMU、角度センサー、手袋スイッチ）で作業者の動作を判定し、駆動部を制御してアシストを行います。システムの構成、状態遷移、リアルタイムOSを用いたタスク設計、そして機能追加（自動追従機能と危険動作検出機能）に関する理解が問われます。

## 出題のポイント
この問題では、組込みシステムの設計・開発において重要となる以下の知識とスキルが問われています。
*   **システム要求仕様の理解**: 提示された図表（外観図、構成図、機能概要、状態遷移図、状態遷移条件一覧など）を正確に読み解き、PAスーツのシステム仕様を詳細に理解する能力。
*   **状態遷移設計**: PAスーツがどのような状態を持ち、どのような条件で状態が遷移するのかを、センサー情報と動作の関係性から正確に把握する能力。特に、IMUや角度センサーの値を姿勢変化と関連付けて理解することが重要です。
*   **リアルタイムOS（RTOS）を用いたタスク設計**: タスク間のメッセージ通信（メールボックス）やタスク優先度を考慮したタスク連携、および処理時間の算出に関する知識。
*   **機能追加・変更への対応**: 新しい機能（自動追従機能、危険動作検出機能）の追加に伴うハードウェアの変更点と、それに合わせたソフトウェアのタスク設計やデータ処理ロジックの変更点を理解し、適切に説明する能力。

#### 設問1(1)(b)
**設問の内容**: 基準タイマーで計測できる競技時間は最大何分となるか。答えは小数点以下を切り捨て、10進数の整数で答えよ。ここで、16ビットで表現できる最大の数は65,535である。

> **正解**: 87

**解答に至るための思考プロセス**:
1.  「**審判計時機のタイマー部**」の仕様を確認します。
    *   「基準タイマーは**20ビット**のカウントアップタイマーで、サブタイマーの値が0になるたびにカウントアップする。」
    *   サブタイマーは320カウントで10ミリ秒なので、基準タイマーの1カウントは10ミリ秒に相当します。
2.  基準タイマーの最大値を特定します。20ビットのタイマーの最大値は $2^{20} - 1 = 1,048,575$ です。
    *   ただし、「競技開始時の故障判定とフライング判定」の(2)で、「基準タイマーの値が**16進数 80000** より大きくなっていることを確認する。」とあり、フライング判定にはこの値が使われています。このため、実質的に基準タイマーがこの値までしか正常にカウントできないと解釈できます。
    *   16進数80000を10進数に変換します。 $8 \times 16^4 = 8 \times 65,536 = 524,288$。
3.  基準タイマーの1カウントは10ミリ秒なので、最大計測可能時間（秒）を計算します。
    *   最大計測可能時間（ミリ秒） = $524,288 \text{カウント} \times 10 \text{ミリ秒/カウント} = 5,242,880 \text{ミリ秒}$。
    *   最大計測可能時間（秒） = $5,242,880 \text{ミリ秒} / 1,000 \text{ミリ秒/秒} = 5,242.88 \text{秒}$。
4.  これを分に変換し、小数点以下を切り捨てます。
    *   最大計測可能時間（分） = $5,242.88 \text{秒} / 60 \text{秒/分} \approx 87.3813 \text{分}$。
    *   小数点以下を切り捨てて、**87分**。

**問題文中の図表やデータの読み取り方**:
*   基準タイマーのビット数とカウント単位、およびフライング判定に用いられる閾値（16進数80000）を正確に読み取ります。
*   タイマーの1カウントあたりの時間と、最大カウント値から最大時間を計算します。

#### 設問3(3)(a)
**設問の内容**: それぞれの処理に最大の処理時間が掛かるとした場合、1回の追跡処理の処理時間は a ミリ秒であり、

> **正解**: 105

**解答に至るための思考プロセス**:
1.  この設問の文脈は「方式2の測定範囲について」であり、`a`、`b`、`c`の相互関係を問うものです。
2.  「1回の追跡処理の処理時間は `a` ミリ秒であり、その時間内に 1 コアが測定可能な範囲は `b` m² である。」という記述から、`b = a / (0.2 ms/m^2)` の関係が成り立ちます。

#### 設問3(3)(b)
**設問の内容**: その時間内に 1 コアが測定可能な範囲は b m² である。

> **正解**: 525

**解答に至るための思考プロセス**:

2.  解答例の`a = 105`を代入すると、$b = 105 / 0.2 = \textbf{525 m}^2$ となります。

**問題文中の図表やデータの読み取り方**:

#### 設問3(3)(c)
**設問の内容**: 方式 2 では一部の範囲を重複して処理するため、追跡ユニットが実際に測定可能な範囲は最大 c m²である。

> **正解**: 1,000

**解答に至るための思考プロセス**:
1.  方式2では「前処理でマッピングした座標空間を二等分に分割して、分割した座標空間ごとにコアを割り当てて追跡処理を並列に実施する」とあります。
2.  さらに「全体の座標空間の5%分を**重複して処理**する必要がある」とあります。
3.  この「全体の座標空間の5%分を重複して処理」という記述は、総処理対象面積が実際の測定範囲Xの1.05倍になる、と解釈するのが最も自然です。
    *   つまり、総処理対象面積 = $1.05 \times X$。
4.  この総処理対象面積を、2つのコアで並列に処理すると考えます。（図3(b)の記述「空間を二等分して並列に実施する」から、2つのコアがこの空間分割を分担すると解釈）
5.  したがって、1つのコアが処理する面積は、総処理対象面積の半分となり、$ (1.05 \times X) / 2 = 0.525 \times X$ となります。
6.  1つのコアが測定可能な最大範囲は`b`m²（設問3(3)(b)で525m²と算出）であるため、以下の関係式が成り立ちます。
    *   $0.525 \times X = b$
    *   $0.525 \times X = 525$
    *   $X = 525 / 0.525 = \textbf{1,000 m}^2$。
7.  「つなぎ合わせ処理として2ミリ秒の処理時間が追加で必要となる」という記述については、採点講評で「別のコアで前処理を行っているタイミングで、つなぎ合わせ処理も実行可能である点を考慮して解答してほしかった」とあり、この処理時間はボトルネックにならず、全体の最大測定範囲には影響しないと解釈できます。

**問題文中の図表やデータの読み取り方**:
*   「空間を二等分して並列に実施」「全体の座標空間の5%分を重複して処理」という記述を、総処理量と各コアの処理分担の関係として正確に解釈することが重要です。特に「重複」の解釈がポイントになります。

**誤答しやすいポイントや注意点**:
*   採点講評では、「1回の追跡処理の処理時間について,つなぎ合わせ処理の処理時間を引いていると思われる解答が一部見受けられた。別のコアで前処理を行っているタイミングで、つなぎ合わせ処理も実行可能である点を考慮して解答してほしかった」とあります。方式2の並列処理では、追加される処理（つなぎ合わせ処理）が他の処理と並行して行われることで、必ずしも全体の処理時間に直接加算されない場合があるというリアルタイムOSの知識が問われています。
