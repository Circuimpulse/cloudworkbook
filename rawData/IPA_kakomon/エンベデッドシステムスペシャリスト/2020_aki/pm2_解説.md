まず、基本的なルールを整理します。
*   白色LEDのLED輝度とオレンジ色LEDのLED輝度を(X, Y)と表記する。
*   白色で最も明るい状態: (255, 0)
*   オレンジ色で最も明るい状態: (0, 255)
*   **色合い変化**: 2種類のLEDを同時に点灯させ、**LED輝度の総和を255に保つ**。
*   **明るさ変化**: その色合いでの最も明るい状態から、指定された割合で下げる。

##### `a` の計算:
*   「白色で最も明るい状態からオレンジ色に1段階近づけた状態」
*   LED輝度の総和は255を維持。
*   「それぞれ最大のLED輝度の1/5ずつ上下する」
    *   白色で最も明るい状態は(255, 0)。
    *   白色LEDは255から1/5下げる → $255 \times (1 - 1/5) = 255 \times 4/5 = 204$
    *   オレンジ色LEDは0から1/5上げる → $255 \times 1/5 = 51$
*   よって、設定値は**(204, 51)**。

##### `b` の計算:
*   「白色で最も明るい状態からオレンジ色に3段階近づけた中間の状態（昼光色相当）」
*   LED輝度の総和は255を維持。
*   3段階近づけるので、白色LEDは255から3/5下げる、オレンジ色LEDは0から3/5上げる。
    *   白色LED: $255 \times (1 - 3/5) = 255 \times 2/5 = 102$
    *   オレンジ色LED: $255 \times 3/5 = 153$
*   よって、設定値は**(102, 153)**。

##### `c` の計算:
*   「白色で最も明るい状態からオレンジ色に3段階近づけた中間の状態（昼光色相当）で明るさを1段階暗くする」
*   まず「昼光色相当」の設定値は `b` で求めた**(102, 153)**。これがこの色合いでの「最も明るい状態」。
*   「明るさを1段階暗くする」場合は「その色合いでの最も明るい状態のLED輝度から1/17ずつ下げればよい」

    *   オレンジ色LED: $153 \times (1 - 1/17) = 153 \times 16/17 = 9 \times 16 = 144$
*   よって、設定値は**(96, 144)**。

これらの値は全て整数であり、四捨五入の必要はありません。

### 設問2 (2)
#### 設問の内容を簡潔に示す
IR通信の信号について、1回目のリーダの送信開始から2回目のデータの送信終了まで、最小及び最大何ミリ秒掛かるか。小数第1位を四捨五入して、整数で答える。

#### 正解
> **正解**:
> 最小: 376
> 最大: 640

#### 解答に至るための思考プロセスと補足説明
問題文の「〔IR通信によるエアコンの操作〕」のセクションと、図6「IR通信の信号の例」およびその注記を確認します。

基本的な情報:
*   1T = 500マイクロ秒 = 0.5ミリ秒
*   IR通信は同じ信号波形を**2回連続して送信**する。

信号波形の内訳（図6から読み取り）：
*   リーダ部: 8T (ON) + 4T (OFF) = 12T
*   データ部: 132ビット
    *   ビット値0: 1T (ON) + 1T (OFF) = 2T
    *   ビット値1: 1T (ON) + 3T (OFF) = 4T
*   トレーラ部: 1T (ON) + 199T (OFF) = 200T

まず、1回分の信号波形の時間を計算します。
1.  **リーダ部の時間**: $12T = 12 \times 0.5 \text{ミリ秒} = 6 \text{ミリ秒}$

2.  **データ部（132ビット）の時間**:
    *   **最小時間**: 132ビット全てがビット値0の場合。
        $132 \times 2T = 264T = 264 \times 0.5 \text{ミリ秒} = 132 \text{ミリ秒}$
    *   **最大時間**: 132ビット全てがビット値1の場合。
        $132 \times 4T = 528T = 528 \times 0.5 \text{ミリ秒} = 264 \text{ミリ秒}$

3.  **トレーラ部の時間**: $200T = 200 \times 0.5 \text{ミリ秒} = 100 \text{ミリ秒}$

1回分の信号波形の合計時間:
*   **最小**: リーダ (6ms) + データ最小 (132ms) + トレーラ (100ms) = $6 + 132 + 100 = 238 \text{ミリ秒}$
*   **最大**: リーダ (6ms) + データ最大 (264ms) + トレーラ (100ms) = $6 + 264 + 100 = 370 \text{ミリ秒}$

設問では「1回目のリーダの送信開始から**2回目のデータの送信終了まで**」の時間を求めます。
IR通信は同じ信号波形を2回連続して送信するので、構成は以下のようになります。
[1回目リーダ] + [1回目データ] + [1回目トレーラ] + [2回目リーダ] + [2回目データ]

*   **最小時間**:
    $6 \text{ミリ秒} (\text{1回目リーダ}) + 132 \text{ミリ秒} (\text{1回目データ}) + 100 \text{ミリ秒} (\text{1回目トレーラ}) + 6 \text{ミリ秒} (\text{2回目リーダ}) + 132 \text{ミリ秒} (\text{2回目データ})$
    $= 6 + 132 + 100 + 6 + 132 = 376 \text{ミリ秒}$

*   **最大時間**:
    $6 \text{ミリ秒} (\text{1回目リーダ}) + 264 \text{ミリ秒} (\text{1回目データ}) + 100 \text{ミリ秒} (\text{1回目トレーラ}) + 6 \text{ミリ秒} (\text{2回目リーダ}) + 264 \text{ミリ秒} (\text{2回目データ})$
    $= 6 + 264 + 100 + 6 + 264 = 640 \text{ミリ秒}$

いずれも整数値なので、四捨五入は不要です。

### 設問2 (4)
#### 設問の内容を簡潔に示す
〔防犯機能〕について、Sドアホン親機に内蔵しているフラッシュメモリに画像データを保存する。MPEG4の圧縮率を,1/600とした場合,最大何分間保存可能か。答えは小数第1位を四捨五入して、整数で求める。ここで,フラッシュメモリの画像データの記憶領域は,480Gバイトとする。また,1kビット=1,000ビット,1Mビット=1,000kビット,1Gビット=1,000Mビットとする。

#### 正解
> **正解**: 80,000

#### 解答に至るための思考プロセスと補足説明
この設問は、与えられたカメラの性能と記憶容量、圧縮率から最大保存時間を計算する問題です。

1.  **1フレームあたりのデータ量（圧縮前）を計算**:
    *   画素数: $2,000 \times 1,000 = 2,000,000$ ピクセル
    *   1画素あたりのデータ量: 24ビット
    *   1フレームあたりのデータ量（圧縮前） = $2,000,000 \text{ピクセル} \times 24 \text{ビット/ピクセル} = 48,000,000 \text{ビット}$

2.  **1フレームあたりのデータ量（圧縮後）を計算**:
    *   MPEG4圧縮率: 1/600
    *   1フレームあたりのデータ量（圧縮後） = $48,000,000 \text{ビット} \times (1/600) = 80,000 \text{ビット}$

3.  **1秒あたりのデータ量（圧縮後）を計算**:
    *   フレームレート: 10フレーム/秒
    *   1秒あたりのデータ量（圧縮後） = $80,000 \text{ビット/フレーム} \times 10 \text{フレーム/秒} = 800,000 \text{ビット/秒}$

4.  **フラッシュメモリの総容量をビット単位に変換**:
    *   記憶領域: 480Gバイト
    *   1Gビット = 1,000Mビット = 1,000,000kビット = 1,000,000,000ビット
    *   1バイト = 8ビット
    *   総容量 = $480 \text{Gバイト} = 480 \times 1,000 \text{Mバイト} = 480 \times 1,000 \times 1,000 \text{kバイト} = 480 \times 1,000 \times 1,000 \times 1,000 \text{バイト}$
    *   総容量 = $480 \times 10^9 \text{バイト} \times 8 \text{ビット/バイト} = 3,840 \times 10^9 \text{ビット}$
    *   $= 3,840,000,000,000 \text{ビット}$

5.  **保存可能な時間（秒）を計算**:
    *   保存時間（秒） = 総容量 / 1秒あたりのデータ量
    *   保存時間（秒） = $3,840,000,000,000 \text{ビット} / 800,000 \text{ビット/秒} = 4,800,000 \text{秒}$

6.  **保存可能な時間（分）に変換**:
    *   保存時間（分） = $4,800,000 \text{秒} / 60 \text{秒/分} = 80,000 \text{分}$

結果は整数値なので、四捨五入は不要です。

#### 解答に至るための思考プロセスと補足説明
この設問は、リアルタイム処理の要件に基づいて必要な転送速度を計算する問題です。

1.  **1フレームあたりのデータ量（圧縮前）を計算**:
    *   画素数: 300,000画素
    *   1画素あたりのデータ量: 3バイト
    *   1フレームあたりのデータ量 = $300,000 \text{画素} \times 3 \text{バイト/画素} = 900,000 \text{バイト}$
    *   ビットに変換: $900,000 \text{バイト} \times 8 \text{ビット/バイト} = 7,200,000 \text{ビット}$

2.  **1フレームの処理サイクル時間を計算**:
    *   フレームレート: 20フレーム/秒
    *   1フレームの処理サイクル時間 = $1 \text{秒} / 20 \text{フレーム} = 0.05 \text{秒} = 50 \text{ミリ秒}$
    *   これは、次のフレームデータが到着するまでの時間です。

3.  **画像データ取込みに許容される時間を計算**:
    *   異常レベルの判定処理と通信処理にかかる最大時間: 10ミリ秒
    *   画像データの取込みは、次のフレームが来る前に完了し、かつ判定処理も完了する必要があります。
    *   画像データ取込みに許容される時間 = 1フレームの処理サイクル時間 - (判定処理 + 通信処理時間)
    *   $= 50 \text{ミリ秒} - 10 \text{ミリ秒} = 40 \text{ミリ秒}$
    *   つまり、1フレームあたり7,200,000ビットのデータを40ミリ秒以内に転送する必要があります。

4.  **必要な転送速度を計算**:
    *   転送速度 = データ量 / 時間
    *   転送速度 = $7,200,000 \text{ビット} / 40 \text{ミリ秒}$
    *   ミリ秒を秒に変換: $40 \text{ミリ秒} = 0.04 \text{秒}$
    *   転送速度 = $7,200,000 \text{ビット} / 0.04 \text{秒} = 180,000,000 \text{ビット/秒}$

5.  **Mビット/秒に変換し、小数第1位を切り上げて整数で求める**:
    *   1Mビット = $10^6$ビットなので、$180,000,000 \text{ビット/秒} / 10^6 = 180 \text{Mビット/秒}$
    *   結果は整数なので、切り上げは不要です。

これはつまり、「**目が閉じた状態の継続時間が、10〜40フレームという範囲に入ることが全くない**」という状況です。

この条件下で「異常レベル1」に移行するには、どうすればよいのか？
唯一の可能性は「1分間に急加速又は急減速を3回検知」だが、設問は「目が閉じているフレームの数」を問うているため、これは関係ない。

**解答例の50フレームに着目する。**
もし目が**50フレーム連続で閉じている**場合、これは2秒（40フレーム）を超えているため、「居眠り」と判定されます（異常レベル3）。
しかし、この「居眠り」と判定された場合でも、その前に「異常レベル2で5秒間解除なし」という条件でレベル3に移行することもあります。

もしかしたら、この問題は、異常レベル1の「眠気と判定」という条件ではなく、**異常レベル1の「眠気」という言葉が示す状態**に到達するのに必要な目の閉じるフレーム数を問うているのかもしれない。
あるいは、単純に「目を閉じた回数」ではなく「**累積した目の閉じたフレーム数**」を問うているのか？
「異常レベル1に移行するまでには、目が閉じているフレームの数は最少幾つ必要か。」

最少、という言葉が重要。
「目が0.5秒以上2秒以下連続して閉じたことを、20秒以内に5回検出すると、眠気と判定する。」
「検出した回数が0回である」
ということは、**10～40フレームの目が閉じる事象は一切なかった**、という意味。
この条件を満たしながら、眠気と判定されるには、目が閉じるフレーム数がどうあるべきか。

実は、「目が**0.5秒未満**で連続して閉じた」事象であれば、検出回数にはカウントされない。
また、「目が**2秒超**で連続して閉じた」事象も、検出回数にはカウントされない。

もし、目が**9フレーム連続で閉じている**事象が20秒以内に5回起きても、これは0.5秒未満なので、「眠気と判定」のカウント対象外。
つまり、いくら9フレーム以下の閉眼を繰り返しても、「眠気と判定」にはならない。

しかし、もし目が**41フレーム連続で閉じている**事象が起きると、これは2秒超なので「居眠り」（異常レベル3）と判定される。
これでレベル3に移行してしまう。問題はレベル1に移行するフレーム数を聞いている。

**唯一の可能性として、設問の「0.5秒以上2秒以下の時間で、目が連続して閉じたことを検出した回数が0回である」という条件は、眠気の判定における「回数条件」が0回だった、と解釈する。**
しかし、この文の記述は「検出した回数が0回」であり、回数条件を満たさなかった、という意味にしか読めない。

**解答例が50であることからの逆算**:
眠気と判定されるための条件は「20秒以内に5回、10フレーム以上40フレーム以下で連続して目が閉じている」こと。
この**「20秒以内」に**、10フレーム以上40フレーム以下の閉眼が**5回検出されると眠気判定**（レベル1）になる。
設問の「検出した回数が0回」という条件は、「**現時点ではまだ0回**」という意味かもしれない。
つまり、この後、眠気と判定されるための5回の検出を達成するのに、最小で何フレームが必要か？
この場合は、**10フレームの閉眼を5回**検出するのが最短。
$10 \text{フレーム} \times 5 \text{回} = 50 \text{フレーム}$
これが最も自然な解釈だ。
「検出した回数が0回である」は「まだ0回だよ」という意味合い。
この後、眠気と判定されるようにするため、最も少ないフレーム数で閉眼を繰り返す。それが10フレームを5回なので、50フレーム。
