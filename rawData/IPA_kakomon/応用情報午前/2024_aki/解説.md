## 問1

**正解: ウ**

**分野**: テクノロジ

**解説**:
M/M/1待ち行列モデルにおける平均待ち時間 $W_q$ は、以下の式で表されます。
$$W_q = \frac{\rho}{\mu(1-\rho)}$$
ここで、$\rho$ は窓口の利用率、$\mu$ は1秒あたりのサービス完了件数（サービス率）です。
サービス率 $\mu$ は窓口の処理能力を示す定数と仮定します。

1.  **利用率25% (0.25) の場合**:
    $$W_{q1} = \frac{0.25}{\mu(1-0.25)} = \frac{0.25}{\mu \times 0.75} = \frac{1}{3\mu}$$

2.  **利用率40% (0.40) の場合**:
    $$W_{q2} = \frac{0.40}{\mu(1-0.40)} = \frac{0.40}{\mu \times 0.60} = \frac{2}{3\mu}$$

平均待ち時間が何倍になるかを計算します。
$$\frac{W_{q2}}{W_{q1}} = \frac{2/(3\mu)}{1/(3\mu)} = \frac{2}{1} = 2$$
したがって、平均待ち時間は2倍になります。

-   **ア**: 1.25倍。計算が誤っています。
-   **イ**: 1.60倍。計算が誤っています。
-   **ウ**: 2.00倍。正しい計算結果です。
-   **エ**: 3.00倍。計算が誤っています。

**キーワード**: **M/M/1待ち行列モデル**、**利用率**、**平均待ち時間**、**サービス率**

**解説**:
**逆ポーランド表記法 (Reverse Polish Notation, RPN)** は、演算子が被演算子の後ろに記述される数式表現形式です。この表記法では、演算子の優先順位や括弧を明示的に記述する必要がなく、スタックを用いて簡単に評価できます。

与えられた式: $A + B \times C$

1.  まず、乗算の方が加算よりも優先順位が高いので、$B \times C$ を先に処理します。
    逆ポーランド表記では $B \ C \times$ となります。

2.  次に、この結果と $A$ を加算します。
    $A$ と $B \ C \times$ の結果に対して加算 ($+$) を適用するので、 $A \ B \ C \times \ +$ となります。

したがって、適切な表現は「ウ ABC×+」です。

-   **ア**: +×CBA。演算子の順番と位置が異なります。
-   **イ**: ×+ABC。演算子の順番と位置が異なります。
-   **ウ**: ABC×+。正しい順序です。
-   **エ**: CBA+×。演算子の順番と位置が異なります。

**キーワード**: **逆ポーランド表記法 (RPN)**、**スタック**、**演算子の優先順位**

## 問6

**正解: イ**

**分野**: テクノロジ

**解説**:
**ハッシュ衝突**とは、異なる2つのキー $a$ と $b$ が、ハッシュ関数 $h(x)$ を通して同じハッシュ値（ハッシュアドレス）を生成してしまう現象です。

与えられたハッシュ関数は $h(x) = x \pmod n$ です。
衝突が発生する条件は、$h(a) = h(b)$、つまり $a \pmod n = b \pmod n$ となることです。

この式は、$a$ を $n$ で割った余りと、$b$ を $n$ で割った余りが等しいことを意味します。
数学的に表現すると、これは $a - b$ が $n$ で割り切れる、すなわち $a - b$ が $n$ の**倍数**であることと同義です。

例: $n = 5$ の場合
*   $a = 7, b = 2$ のとき
    $h(7) = 7 \pmod 5 = 2$
    $h(2) = 2 \pmod 5 = 2$
    ハッシュ値が同じで衝突します。このとき、$a - b = 7 - 2 = 5$ であり、$5$ は $n$ ($5$) の倍数です。

-   **ア**: a+bがnの倍数。誤りです。例えば $a=1, b=2, n=3$ の場合、$a+b=3$ で $n$ の倍数ですが、$1 \pmod 3 = 1$、$2 \pmod 3 = 2$ で衝突しません。
-   **イ**: a-bがnの倍数。正しいです。ハッシュ値が等しい場合、$a \pmod n = b \pmod n$ なので、$a-b$ は $n$ の倍数になります。
-   **ウ**: nがa+bの倍数。誤りです。
-   **エ**: nがa-bの倍数。誤りです。a-bがnの倍数です。

**キーワード**: **ハッシュ関数**、**ハッシュ衝突**、**剰余演算 (mod)**

**RISCプロセッサ**の5段パイプラインにおける命令実行制御の標準的な順序は以下の通りです。

1.  **命令フェッチ (Instruction Fetch, IF)**: メモリから命令を読み出す。 (問題文の④)
2.  **命令デコード／レジスタフェッチ (Instruction Decode/Register Fetch, ID)**: 読み出した命令を解読し、必要なオペランドをレジスタファイルから読み出す。 (問題文の③)

5.  **ライトバック (Write Back, WB)**: 演算結果やメモリから読み出したデータをレジスタファイルに書き戻す。 (問題文の①)

-   **ウ**: (4), (3), (2), (5), (1)。正しい順序です。

**正解: イ**

**分野**: テクノロジ

**解説**:
**実効メモリアクセス時間**は、キャッシュメモリと主記憶のアクセス時間、そしてキャッシュのヒット率から計算されます。

与えられた条件:
*   主記憶のアクセス時間 $T_M$
*   キャッシュメモリのアクセス時間 $T_C = T_M / 30$
*   ヒット率 $h = 0.95$

実効メモリアクセス時間 $T_E$ の計算式は、キャッシュヒット時とキャッシュミス時の平均として表されます。
$$T_E = h \times T_C + (1 - h) \times (T_C + T_M)$$
ここで、
*   キャッシュヒット時: $T_C$ の時間でアクセス完了。
*   キャッシュミス時: $T_C$ の時間でキャッシュを検索し、その後 $T_M$ の時間で主記憶にアクセス。合計 $T_C + T_M$。

これらの値を式に代入します。
$$T_E = 0.95 \times \frac{T_M}{30} + (1 - 0.95) \times \left(\frac{T_M}{30} + T_M\right)$$
$$T_E = 0.95 \times \frac{T_M}{30} + 0.05 \times \left(\frac{T_M}{30} + \frac{30T_M}{30}\right)$$
$$T_E = 0.95 \times \frac{T_M}{30} + 0.05 \times \frac{31T_M}{30}$$
$$T_E = \frac{T_M}{30} \times (0.95 + 0.05 \times 31)$$
$$T_E = \frac{T_M}{30} \times (0.95 + 1.55)$$
$$T_E = \frac{T_M}{30} \times 2.50$$
$$T_E = \frac{2.50}{30} \times T_M$$
$$T_E = \frac{25}{300} \times T_M = \frac{1}{12} \times T_M$$
$1 \div 12 \approx 0.0833...$

主記憶のアクセス時間の約 $0.08$ 倍となります。

-   **ア**: 0.03。
-   **イ**: 0.08。最も近い値です。
-   **ウ**: 0.37。
-   **エ**: 0.95。

**キーワード**: **キャッシュメモリ**、**主記憶**、**ヒット率**、**実効メモリアクセス時間**

    システムが稼働するには、両方の装置が稼働している必要があります。
    稼働率 $R_A = r \times r = r^2$

2.  **システムB**: 装置Xが2つ並列に接続されています。
    システムが稼働するには、どちらか一方の装置が稼働していればよいです。
    稼働率 $R_B = 1 - (1-r) \times (1-r) = 1 - (1-r)^2 = 1 - (1 - 2r + r^2) = 2r - r^2$

3.  **システムC**: (装置Xが2つ直列) のシステムが2つ並列に接続されています。

**正解: イ**

**分野**: テクノロジ

**解説**:
この問題は、SQLの**ウィンドウ関数 (Window Function)** を用いて、特定の条件を満たす行を選択する方法に関するものです。

**SQL文1の分析**:
```sql
SELECT R1.学生番号, R1.実施回, R1.得点
FROM 成績 R1
INNER JOIN (SELECT 学生番号, MIN(実施回) AS 初回 FROM 成績 GROUP BY 学生番号) R2
ON R1.学生番号 = R2.学生番号 AND R1.実施回 = R2.初回
```
SQL文1は、サブクエリで各学生の**最も早い実施回**（`MIN(実施回)`）を特定し、その情報を使って元の「成績」表から、各学生の初回実施回のレコードを選択しています。
つまり、目的は「**各学生の初回実施回の成績を取得する**」ことです。

**SQL文2の分析**:
```sql
SELECT 学生番号, 実施回, 得点
FROM (
  SELECT 学生番号, 実施回, 得点,
         ROW_NUMBER() OVER (a) AS 番号
  FROM 成績
) R1
WHERE R1.番号 = 1
```
SQL文2では、`ROW_NUMBER()` というウィンドウ関数を使用しています。
`ROW_NUMBER()` 関数は、`OVER`句で指定された**パーティション（グループ）** 内で、指定された**順序**に基づいて各行に一意の連番を割り振ります。
`WHERE R1.番号 = 1` は、各パーティションの最初の行（番号が1の行）のみを選択することを意味します。

SQL文1と同じ結果（各学生の初回実施回の成績）を得るためには、`ROW_NUMBER()` 関数を以下の条件で適用する必要があります。
1.  **学生番号ごとにパーティションを分ける**（グループ化する）: `PARTITION BY 学生番号`
2.  各パーティション内で、**実施回が最も早いものに1番の番号を割り振る**ために、実施回で昇順に並べ替える: `ORDER BY 実施回 ASC` (ASCは昇順で、省略可能です)

したがって、`a`に入れるべき字句は「`PARTITION BY 学生番号 ORDER BY 実施回`」となります。

-   **ア**: `ORDER BY 学生番号, 実施回`。パーティションを分けずに全体をソートするだけなので、各学生の初回実施回だけを選択できません。
-   **イ**: `PARTITION BY 学生番号 ORDER BY 実施回`。学生番号でパーティション化し、各パーティション内で実施回を昇順にソートすることで、各学生の初回実施回が番号1になります。これが正解です。
-   **ウ**: `PARTITION BY 学生番号 ORDER BY 得点 ASC`。得点順でソートするため、初回実施回とは関係ありません。
-   **エ**: `PARTITION BY 学生番号 ORDER BY 得点 DESC`。得点順でソートするため、初回実施回とは関係ありません。

**正解: ウ**

**分野**: テクノロジ

**解説**:
1Mバイトのデータを5秒以内に送信するための、最低速度の回線サービスを計算します。伝送制御のための情報は考慮せず、伝送効率は60%とします。

1.  **送信するデータ量 (ビット単位)**:
    *   1 Mバイト = $1 \times 1024 \times 1024$ バイト = $1,048,576$ バイト
    *   1 バイト = 8 ビット
    *   データ量 = $1,048,576 \text{ バイト} \times 8 \text{ ビット/バイト} = 8,388,608 \text{ ビット}$

2.  **必要な実効伝送速度**:
    *   送信時間: 5秒以内
    *   必要な実効伝送速度 = $\frac{\text{データ量}}{\text{送信時間}} = \frac{8,388,608 \text{ ビット}}{5 \text{ 秒}} = 1,677,721.6 \text{ ビット/秒}$

3.  **回線の最低速度 (契約速度)**:
    *   回線速度（契約速度）を $R$ とすると、実効伝送速度は $R \times \text{伝送効率}$ で計算されます。
    *   $1,677,721.6 \text{ ビット/秒} = R \times 0.6$
    *   $R = \frac{1,677,721.6}{0.6} \approx 2,796,202.67 \text{ ビット/秒}$

4.  **Mビット/秒 (Mbps) への変換**:
    *   $1 \text{ Mビット} = 10^6 \text{ ビット}$
    *   $R \approx 2.796 \text{ Mビット/秒}$

この速度を満たす最低の回線サービスを選択肢から選びます。
-   **ア**: 1 Mビット/秒 (不足)
