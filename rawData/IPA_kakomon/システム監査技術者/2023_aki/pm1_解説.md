# 問2 解説

### 問題の概要
本問題は、C社が業務効率向上のため、利用部門主体でローコード/ノーコード開発ツール（以下、開発ツール）を用いたアプリ開発を行っている状況がテーマです。CIOは管理ルールの不在による保守リスクを懸念し、システム部に管理ルール作成を指示。監査部は、作成された管理ルール案がアプリ開発のリスクを低減できるかどうかを監査する、というシナリオです。予備調査の結果と監査手続書の検討内容が示され、各設問ではリスク識別や監査手続の内容が問われます。

### 出題のポイント
この問題では、**ローコード/ノーコード開発**の特性（利点と欠点）を理解し、それに伴う**システム開発のリスク**（特に管理面、セキュリティ面、保守面）を識別する能力が問われています。また、識別されたリスクに対して**どのようなコントロールが必要か**、そしてそれを確認するための**具体的な監査手続を立案する能力**が重要となります。管理ルール案の適用前段階の監査であるため、運用前のルール自体の評価が中心となります。

### 設問ごとの解説

2.  **関連する人事部インタビュー結果の確認**: 人事部へのインタビュー(1)「人事情報管理」において、「人事異動時には、人事部がスプレッドシートで作成した“異動管理表”のデータを人材管理システムへアップロードし、“基本情報”を更新する。現在はアップロードの前に“異動管理表”へ上長の役職と氏名を手作業で入力しており、その作業に**数日を要している**」と記載されています。
3.  **基本情報の閲覧権限**: 同じく人事部インタビュー(1)で、「各従業員の部署・役職・連絡先は**全ての従業員が閲覧できたが**、人材管理システム再構築によって、各従業員の上長の役職と氏名についても、“基本情報”として、**全ての従業員が閲覧できるようになった**」とあります。
4.  **リスクの特定**: これらの情報から、多数の従業員が異動した場合、「基本情報」の更新に数日かかるため、異動後すぐに新しい部署の役職や上長の情報に更新されない可能性があります。その間、**異動前の部署の従業員が、異動後も閲覧権限を保持している状態**が続く、あるいは**異動先の従業員が、異動前の権限で異動先の情報を閲覧できる**といった状況が生じる可能性があります。これは、本来であれば参照権限が変更されるべき従業員が、古い権限で新しい、あるいはもはや関係のない情報を参照できるというリスクにつながります。特に、人事情報（部署、役職、上長氏名など）の閲覧権限が「全ての従業員」に付与されているため、このリスクは大きいです。

**関連する重要な技術知識・用語の補足説明**:
*   **アクセス権限**: 情報資源に対するユーザーの操作（閲覧、書き込み、削除など）を許可または制限する設定。
*   **職務分離**: 組織内で不正や誤謬を防ぐために、特定の権限を分散させること。人事情報管理においては、情報の閲覧・変更権限は職務に応じて厳密に管理されるべきです。

**誤答しやすいポイントや注意点（落とし穴）**:
*   採点講評でも「単に人事情報の閲覧というだけではなく,人事異動に伴う閲覧権限の変更を理解して,リスクを明確に捉えてほしい」とあるように、単に「情報が古い」というだけでなく、それが「閲覧権限の不整合」を通じて「不正な情報閲覧」につながるリスクである点を明確にすることが重要です。

#### 設問6
設問の内容を簡潔に示す: 〔本調査のための検討〕 (4)について、監査チームが確認すべき内容を30字以内で答えよ。

> **正解**: 各職場で増えた作業も考慮して業務効率を算定していること

**解答に至るための思考プロセス**:
1.  **検討事項(4)の内容確認**: 「業務効率向上の効果が過大に算定されている可能性があるので、“利用状況報告書”を閲覧して効果の算定内容を確認する。」とあります。
2.  **「業務効率向上の効果が過大算定」の原因**: 情報システム部インタビュー(3)で、「従来は人事部を介して行っていた人材管理業務の多くが、人材管理システムの再構築後は、各職場の管理者や従業員が自ら行えるようになった。その結果,便利になったこともあるが、一方で**人事に関わる情報の入力やその確認など、新たに各職場で行うことになった作業も多いと感じている**」と記載されています。
3.  **効果算定の課題**: システム導入によって一部の業務が効率化されても、その業務が他の部署に移管され、そこで新たな作業負担が生じている場合、組織全体の業務効率が必ずしも向上しているとは限りません。全体最適の観点から、こうした**移管された業務による負担増も考慮に入れて**、総合的な業務効率を算定する必要があります。
4.  **確認すべき内容**: したがって、監査チームは、「**各職場で増えた作業も考慮して業務効率を算定していること**」を確認すべきです。

**関連する重要な技術知識・用語の補足説明**:
*   **システム導入効果測定**: システム導入の目的が達成されたか、どの程度の効果があったかを定量的に評価すること。部分的な効果だけでなく、組織全体の視点から評価することが重要です。
*   **TCO (Total Cost of Ownership)**: システム導入後の運用、保守、管理などを含めた総保有コスト。効果測定においても、単なる開発コストだけでなく、間接的な運用コストや労力増減も考慮することが重要です。

**誤答しやすいポイントや注意点（落とし穴）**:
*   採点講評で「業務効率向上の効果算定は,システム導入において不可欠な活動であり,その監査の観点はよく理解しておいてほしい」とあるように、システム導入効果の評価はシステム監査の重要なポイントです。
*   単に「効率化効果が適切か」という漠然とした内容ではなく、問題文中で具体的に指摘されている「各職場で増えた作業」という負の側面も考慮に入れているかを問うことが重要です。

### 採点講評からのアドバイス
問3全体として、人材管理システムの有効活用とそれに伴う新たな課題やリスクを理解し、システム監査を実施する能力が問われています。

*   **設問1**: 「人事異動に伴う閲覧権限の変更」という動的な側面と、それが引き起こす情報閲覧のリスクを明確に捉えることが重要です。単なる情報鮮度の問題ではなく、セキュリティリスクとして記述する訓練が必要です。
*   **設問3**: データ連携における「正確性」と「網羅性」の違いを理解し、特に「網羅性」を確保するための具体的な統制（照合作業）が手順書に記載されているかを確認する視点を持つことが重要です。
*   **設問5**: 管理者の「評価」という**動機**が、システムの不適切な利用（適合しない人材の登録）を誘発するリスクを識別できるかがポイントです。単なる操作ミスではなく、意図的な不正につながる可能性を考慮したリスク分析の視点を養いましょう。
*   **設問6**: システム導入の効果測定において、**全体最適の視点**が重要です。一部の効率化だけでなく、他の部門に転嫁された作業負担なども含めた総合的な視点で業務効率が算定されているかを確認する能力が求められます。

人材管理システムは個人情報や機微な情報を扱うため、セキュリティとガバナンスが特に重要です。システムの機能だけでなく、それが組織の業務プロセスや従業員の行動にどのような影響を与えるかを総合的に分析する力を養いましょう。
