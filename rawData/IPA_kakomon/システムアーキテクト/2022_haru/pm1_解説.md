# 問1 解説

## 問題の概要
化粧品や健康食品の製造・販売を行うA社が、現在のコンタクトセンタにおける様々な課題を解決するため、新たなコンタクトセンタシステムを構築するプロジェクトに関する問題です。電話とWebフォーム中心の既存サービスから、AIチャットボット、有人チャット、ボイスボット、ビデオ通話などの**オムニチャネル対応**や、**クラウド型PBX**、**ナレッジベース**、**キーワード分析**といった最新技術を導入し、顧客サービスの向上と業務効率化を目指します。

## 出題のポイント
この問題では、企業の**カスタマサービス**におけるデジタル技術の活用に関する深い理解が問われます。具体的には、
- **現行業務の課題分析**から、新システムに必要な**機能要件**や**非機能要件**を導出する能力。
- 新しいテクノロジー（AIチャットボット、ボイスボット、クラウド型PBXなど）が、どのようなビジネス課題を解決し、どのような**運用設計**に結びつくかを理解する能力。
- 問題文中の複数の表（顧客の行動と問合せ内容、カスタマサービスの課題、新システムのサービス機能）と本文の記述を総合的に読み解き、関連付けて解答を導く**情報分析力**。
- **システムアーキテクト**として、要求事項をシステム設計に落とし込む実践的なスキル。

### 設問1(1)
**設問の内容**: クラウド型 PBX を導入した、オペレータの勤務形態の改善に関する目的を25字以内で述べよ。
**正解**:

**解答に至るための思考プロセス**:
1.  まず、設問が「勤務形態の改善に関する目的」を尋ねていることに注目します。
2.  問題文の「〔カスタマサービスの課題〕」セクションにある**表2「カスタマサービスの課題」**を確認します。
3.  表2の「対象者：コンタクトセンタの管理者」の項目(e)に、「近年は各地域内でコンタクトセンタの設置が増えており、オペレータの人材確保が困難になっている。一方で、育児、介護などで、フルタイムで出社して働くことが難しいオペレータもおり、**在宅かつ柔軟な勤務時間で働きたいというニーズが高まっている**。」と記載されています。これが勤務形態に関する課題とニーズです。
4.  次に、「〔新たなコンタクトセンタシステムの構築〕」セクションにある**表3「新たなコンタクトセンタシステムのサービス機能」**を確認します。
5.  「**クラウド型 PBX**」の機能概要には、「オペレータが利用する電話機を PC 上で電話対応できるソフトフォンに変更し、PCからインターネットを経由して電話の発着信,通話などができるようにする。」とあります。
6.  これらの情報を総合すると、クラウド型PBXの導入により、オペレータが物理的なコンタクトセンタに出社せずとも、PCとインターネットがあれば電話対応が可能になるため、上記(e)のニーズである「在宅勤務」が実現できることが分かります。
7.  これを25字以内で簡潔にまとめると、「オペレータが在宅でも働けるようにするため」となります。

**関連する重要な技術知識・用語の補足説明**:
-   **PBX (Private Branch eXchange)**: 構内交換機。企業内の電話網を制御し、外線との接続や内線通話、転送などの機能を提供します。
-   **クラウド型PBX**: PBXの機能をクラウドサービスとして提供するもので、物理的な機器を社内に設置する必要がありません。インターネット環境があれば、場所を問わず電話機能を利用できるため、**在宅勤務**や**多拠点オフィス**での利用に柔軟に対応できます。

**誤答しやすいポイントや注意点（落とし穴）**:
-   単に「オペレータの業務を効率化するため」といった一般的な表現では、勤務形態の改善という設問の意図に合致しません。表2(e)に示された具体的なニーズに着目することが重要です。

### 設問1(2)
**設問の内容**: AI チャットボット及びボイスボットでは、稼働当初はどのような問合せに対応することを想定しているか。問合せ時の顧客の行動を表1中から全て答えよ。
**正解**:

**解答に至るための思考プロセス**:
1.  まず、AIチャットボットとボイスボットの「稼働当初の対応想定」という条件に注目します。
2.  「〔新たなコンタクトセンタシステムの構築〕」セクションの「AI チャットボット及びボイスボットは、将来的に広範囲の問合せに対応できることを目指すが、有効性を評価しながら段階的に対象を増やすことにする。稼働当初は、オペレータの業務を補完する目的で、**急増しやすい問合せ、かつそれぞれの機能の特徴で対応できる見込みが高い問合せ**を対象にする。一方で、新商品購入者の声を直接聞きたい狙いもあり、顧客が**使用中の商品に関する問合せは、稼働当初は対象にしない**。」という記述を注意深く読み込みます。
3.  この記述から、以下の3つの条件を満たす問合せを探します。
    -   (a) 急増しやすい問合せ
    -   (b) AI機能で対応できる見込みが高い（＝定型的、情報提供、簡単な手続きなど）
    -   (c) 「使用中の商品に関する問合せ」ではない
4.  **表1「顧客の行動ごとのよくある問合せ内容及び特徴」**を各項目で評価します。
    -   **商品購入前の検討**: 「体質など顧客ごとに気になる点が異なり、多岐にわたる傾向がある。」とあり、AIでの定型的な対応は難しいと判断できます（表2(d)の課題「対応時間が長くなる」とも関連）。→ 条件(b)に合致しないため除外。
    -   **購入方法の情報収集**: 「店舗情報を参照し、明確に回答できるものが多い。」とあり、定型的な情報提供に適しています。表2(a)(g)の課題「問合せが急増する」「FAQを早く掲載したい」とも関連し、急増する可能性もあります。→ 条件(a)(b)(c)に合致。
    -   **オンラインストアでの購入**: 「配送業者との調整など、オンラインストア上で処理できないものが多い。」とあり、複雑な手続きや外部連携が必要なケースも含まれるため、AIでの「対応できる見込みが高い」とは言い切れません。→ 条件(b)に合致しない可能性。
    -   **購入した商品の使用**: 「使用順序,タイミング」など。これは「使用中の商品に関する問合せ」に該当するため、稼働当初は対象外です。→ 条件(c)に合致しないため除外。
    -   **会員情報の確認**: 「本人確認を行うことで、手続、回答できるものが多い。」とあり、定型的な手続きや情報提供に適しています。表2(b)の課題「会員情報に関する問合せが急増し、1件ごとの対応は簡単であるが、1日中電話が鳴りやまない」とも関連しており、急増しやすい問合せです。→ 条件(a)(b)(c)に合致。
5.  以上の評価から、「購入方法の情報収集」と「会員情報の確認」が該当すると判断できます。

**関連する重要な技術知識・用語の補足説明**:
-   **AIチャットボット/ボイスボット**: 自然言語処理技術を用いて、顧客からの問い合わせを理解し、事前に登録されたFAQやシナリオに基づいて自動で回答や手続きを行うシステムです。定型的な問い合わせ対応や情報提供に強みがあります。

**誤答しやすいポイントや注意点（落とし穴）**:
-   問題文中の「**稼働当初は対象にしない**」という条件（「使用中の商品に関する問合せ」）を見落とすと誤答につながります。
-   AIの特性（定型的な対応が得意、複雑な判断は苦手）を理解し、表1の「問合せ内容の特徴」と照らし合わせることが重要です。採点講評でも「サービス導入の方針をよく理解して,正答を導き出してほしい」と指摘されています。

**設問の内容**: 電話のコールバックの仕組みを導入して解決を図る直接的な課題を表2中の(a)~(j)の記号で一つ答えよ。

    -   (b) オンラインストアの会員限定セール・キャンペーン時の会員情報問合せ急増。

-   **コールバック**: 顧客が電話で待機することなく、空きができたオペレータから後で電話をかけ直してもらうサービスです。顧客の待ち時間ストレスを軽減し、**顧客満足度**向上に貢献します。

### 設問2(2)
**設問の内容**: 点検制御装置がカメラロボットの運用を安定的に行うために、風向・風力計のデータを活用したい。診断サーバに蓄積されているデータを使用する場合の問題点を40字以内、その解決策を35字以内で述べよ。
**正解**:
> **問題点**: **データがリアルタイムでないため,カメラロボットの飛行制御には適さない**
> **解決策**: **必要なデータを診断サーバを経由してオンデマンドで取得する。**

**解答に至るための思考プロセス**:
1.  まず、「**風向・風力計のデータ**を活用してカメラロボットの運用を安定化する」という目的と、「診断サーバに**蓄積されているデータを使用する場合の問題点**」および「**解決策**」という設問の意図に注目します。
2.  カメラロボットの飛行制御において、風向・風力は刻々と変化する環境要因であり、安定的な飛行には**リアルタイム性の高いデータ**が必要です。
3.  現行システムでは、センサノードは「1時間ごとに診断サーバに送信する」とあります。診断サーバに「蓄積されている」データは、過去のデータであり、1時間前のデータではリアルタイムの風向・風力には対応できません。これが「**問題点**」です。
4.  次に解決策を考えます。現行システムのセンサノードの機能概要には、「診断サーバからの要求を受け取ると、**オンデマンドでセンサデータを、ゲートウェイを介して、診断サーバに送信することもできる。**」とあります。新システムのセンサノードも同様に「診断サーバからの要求によって**オンデマンドで計測を行うことが可能**である」と記載されています。
5.  この「オンデマンドでのデータ取得」機能を利用することで、必要な時に最新の風向・風力データを取得し、リアルタイムに近い情報で飛行制御を行うことが可能になります。
6.  それぞれ字数制限内で簡潔にまとめます。

**関連する重要な技術知識・用語の補足説明**:
-   **リアルタイム処理**: データが発生した直後に、ごく短い遅延で処理が実行されるシステムのことです。ロボットの飛行制御や産業用制御システムなど、時間的な制約が厳しい分野で不可欠な要素です。
-   **オンデマンド**: ユーザーやシステムからの要求に応じて、必要なデータやサービスが提供される方式です。

**誤答しやすいポイントや注意点（落とし穴）**:
-   問題点を「データの標準化がされていないこと」と誤解しないように注意しましょう。風向・風力計のデータはリアルタイム性が生命線であり、この点に着目することが重要です。採点講評でも「風向・風力のデータを基にカメラロボットを制御するためには、データのリアルタイム性が重要であることを理解してほしい。」と明確に指摘されています。
